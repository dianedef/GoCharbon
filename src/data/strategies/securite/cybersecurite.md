# Cybersécurité : le minimum sérieux qu'une petite entreprise doit mettre en place

La cybersécurité paraît abstraite jusqu'au jour où un accès est compromis, un site tombe, un compte pub disparaît ou des données client partent dans la nature. À partir de là, ça devient soudain très concret.

## Le vrai problème des petites structures

Elles pensent souvent être trop petites pour être ciblées. En pratique, beaucoup d'attaques sont opportunistes : mots de passe faibles, plugins non mis à jour, accès partagés, absence de double authentification, sauvegardes inexistantes.

## Les protections essentielles

### Sécuriser les accès

- mots de passe uniques ;
- gestionnaire de mots de passe ;
- double authentification ;
- revue régulière des comptes et droits.

### Sauvegarder correctement

- sauvegardes automatiques ;
- copie hors du serveur principal ;
- tests de restauration.

### Mettre à jour

CMS, plugins, dépendances, serveurs, postes de travail. Une faille connue non corrigée reste l'une des portes d'entrée les plus bêtes et les plus fréquentes.

### Réduire l'exposition

- moins de comptes admin ;
- moins d'outils connectés inutilement ;
- moins de scripts et services oubliés ;
- moins de partage sauvage d'identifiants.

## Ce qu'il faut préparer à l'avance

- qui contacte qui en cas d'incident ;
- où se trouvent les sauvegardes ;
- quels accès couper en premier ;
- comment informer les clients si nécessaire ;
- quels prestataires peuvent aider rapidement.

Le rapport 2025 du [Verizon DBIR](https://www.verizon.com/business/resources/reports/dbir/) reste une bonne piqûre de rappel : erreurs humaines, identifiants compromis et exploitation de vulnérabilités connues restent parmi les causes les plus fréquentes d'incident.

## Le niveau de maturité réaliste

Tu n'as pas besoin d'une cellule cyber pour être sérieux. Tu as besoin d'un minimum propre, appliqué de façon régulière. La sécurité parfaite n'existe pas. L'hygiène, elle, existe. Et elle évite déjà beaucoup de dégâts évitables.
