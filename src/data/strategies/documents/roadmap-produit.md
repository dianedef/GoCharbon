# Roadmap produit : prioriser sans transformer le backlog en décharge

Une roadmap produit n'est pas un cimetière à idées ni une promesse commerciale gravée pour six mois. C'est un outil de priorisation.

## Ce qu'elle doit aider à faire

- choisir l'ordre des chantiers ;
- relier les priorités à un objectif ;
- rendre visibles les arbitrages ;
- éviter de travailler sur dix sujets à la fois.

## Les formats qui marchent bien

- `Now / Next / Later` pour les petites équipes ;
- horizon trimestriel pour les projets plus structurés ;
- approche par objectifs quand le résultat attendu est bien défini.

## Les erreurs fréquentes

- lister des features sans expliquer pourquoi ;
- mettre tout en priorité haute ;
- ne jamais retirer d'items ;
- figer la roadmap alors que la réalité bouge.

Une roadmap utile ne donne pas seulement envie d'ajouter. Elle aide surtout à dire non.
