# Executive summary : résumer un projet pour donner envie d'aller plus loin

L'executive summary est souvent le seul document lu en entier avant une décision de rendez-vous, d'étude ou de financement. Son rôle n'est pas de tout raconter. Son rôle est de rendre le projet suffisamment clair et crédible pour que la conversation continue.

## Ce qu'il doit contenir

- le problème ;
- la solution ;
- le marché ;
- le modèle économique ;
- la traction ou les premiers signaux ;
- l'équipe ;
- la demande éventuelle.

## Ce qu'il ne doit pas devenir

- une introduction molle ;
- un mini business plan de quatre pages ;
- une succession d'adjectifs ;
- un document où les chiffres sont absents ou invérifiables.

## La bonne logique

Chaque bloc doit aider le lecteur à répondre à une question :

- le problème est-il réel ;
- la solution est-elle crédible ;
- le marché vaut-il quelque chose ;
- les porteurs sont-ils sérieux ;
- y a-t-il des signaux de réalité ;
- qu'attend-on précisément de moi.

## Règles utiles

- une page si possible ;
- des chiffres concrets plutôt que des mots vagues ;
- un langage compréhensible sans contexte ;
- une demande explicite à la fin.

L'executive summary n'est pas là pour impressionner. Il est là pour faire avancer la décision.
