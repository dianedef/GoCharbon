# One-pager : faire comprendre un projet en une page nette

Quand quelqu'un te dit "envoie-moi un truc pour comprendre", tu n'as pas besoin d'un deck de 18 slides. Tu as besoin d'une page claire.

## Le contenu minimum

- le problème ;
- la solution ;
- la cible ;
- le fonctionnement en quelques points ;
- la différence principale ;
- un appel à l'action.

## Les règles simples

- une page ;
- peu de jargon ;
- un message immédiatement compréhensible ;
- un seul objectif de lecture.

## Les erreurs classiques

- parler de soi au lieu du client ;
- ajouter trop de texte ;
- vouloir convaincre tout le monde à la fois ;
- faire un document qui ressemble à un résumé de site web.

Le one-pager n'est pas là pour tout dire. Il est là pour donner envie d'en parler.
