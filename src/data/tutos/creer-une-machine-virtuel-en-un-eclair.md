Une machine virtuelle sert à une chose simple : avoir un environnement séparé pour tester, héberger ou expérimenter sans casser le reste.

## Deux options

### VM dans le cloud

Le bon choix si tu veux :

- un accès distant ;
- une IP publique ;
- un serveur jetable ;
- un environnement prêt vite.

### VM en local

Le bon choix si tu veux :

- tester sans coût récurrent ;
- isoler un système ;
- faire du labo perso.

## Les choix à faire

- système d’exploitation ;
- taille minimale ;
- accès SSH ou mot de passe ;
- sauvegarde éventuelle ;
- ouverture réseau nécessaire.

## Bonne pratique

Commence petit :

- 1 vCPU ;
- peu de RAM ;
- une distribution stable ;
- un accès SSH propre.

Tu pourras toujours monter ensuite.

Le plus important n’est pas de créer la VM en trente secondes. C’est d’éviter la VM oubliée, mal sécurisée, et jamais nettoyée.
