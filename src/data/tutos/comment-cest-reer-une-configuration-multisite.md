# Comment créer une configuration WordPress multisite

Si tu veux créer une configuration WordPress multisite, fais simple. Sur WordPress, la plupart des galères viennent d'une mauvaise préparation, d'un plugin en trop ou d'un test final oublié.

## Ce qu'il faut préparer

- une sauvegarde
- un accès admin propre
- un environnement de test si le site est en production

## La méthode simple

1. Commence par la sauvegarde et le contexte. Beaucoup de tutoriels oublient cette étape, puis tout le monde panique quand un plugin casse le site.
2. Applique le changement avec le moins d'intermédiaires possible : un réglage natif, sinon un plugin simple et maintenu, pas un empilement exotique.
3. Teste ensuite le front, l'admin et le mobile. Sur WordPress, une modification 'réussie' dans le back office peut rester cassée côté visiteur.

## Les erreurs à éviter

- faire le changement en prod sans sauvegarde
- installer un plugin de plus alors que WordPress sait déjà le faire
- oublier le cache et les tests sur mobile

## Commence comme ça

Prends un site de test ou un créneau calme, fais une sauvegarde, applique le changement, puis vérifie le front avant de considérer le job terminé.
