# Construire un tableau de bord personnalisé sur WordPress

Le tableau de bord WordPress par défaut est souvent trop technique pour un client ou une équipe non experte.

Créer un dashboard personnalisé permet de:
- guider les utilisateurs,
- réduire les erreurs,
- centraliser la documentation et le support.

## Objectif

Obtenir un espace admin clair avec:
- message d'accueil,
- liens d'aide,
- actions fréquentes,
- formulaire de support.

## Étape 1 - Définis les besoins par rôle

Liste les rôles:
- admin,
- éditeur,
- auteur,
- contributeur,
- client.

Pour chaque rôle, précise:
- ce qu'il doit faire,
- ce qu'il ne doit pas toucher.

## Étape 2 - Prépare une page Dashboard dédiée

Crée une page WordPress "Dashboard" avec:
- bloc bienvenue,
- checklist rapide,
- liens vers tutoriels internes,
- section support.

Tu peux la construire avec Gutenberg ou ton builder.

## Étape 3 - Nettoie le menu admin

Via plugin d'admin customisation (ex: WP Admin Cleaner ou équivalent):
- masque les menus inutiles,
- réordonne les menus,
- adapte par rôle.

But: afficher seulement l'essentiel.

## Étape 4 - Ajoute de l'aide contextuelle

Exemples utiles:
- "Comment publier un article"
- "Comment ajouter un média"
- "Qui contacter en cas de blocage"

Évite les longs pavés: préfère des guides courts et actionnables.

## Étape 5 - Ajoute un canal support

Le minimum:
- un formulaire interne,
- un email support,
- un délai de réponse annoncé.

## Étape 6 - Sécurise l'expérience

- retire l'accès aux réglages critiques pour les profils non techniques,
- force HTTPS,
- limite les plugins visibles,
- journalise les actions sensibles.

## KPI à suivre

- nombre de tickets support liés au back-office,
- temps moyen pour publier un contenu,
- erreurs de manipulation,
- satisfaction des utilisateurs internes.

## Erreurs fréquentes

- dashboard joli mais non utile,
- trop d'options visibles,
- pas de documentation,
- pas de distinction par rôle.

## Plan 5 jours

### Jour 1
Audit des rôles et besoins.

### Jour 2
Création page dashboard.

### Jour 3
Nettoyage menu admin.

### Jour 4
Ajout aide + support.

### Jour 5
Tests utilisateurs et ajustements.

## Checklist

- [ ] rôles définis
- [ ] dashboard créé
- [ ] menus simplifiés
- [ ] aide intégrée
- [ ] support activé
- [ ] tests validés
