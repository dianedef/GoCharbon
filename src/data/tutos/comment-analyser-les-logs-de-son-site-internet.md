### Les bases : D’où viennent les logs ?

Dès qu'un visiteur ou dès qu'un robot vient visiter une page sur un site internet il va télécharger le code source, pour ça :

- Il parse le code source et détecte tous les éléments dont il a besoin pour faire le rendu final de la page : Ca peut être une police de caractères, des fichiers css, des images…léchées javascript etc
- Il va les demander derrière aux serveurs
- Donc il ne charge pas que la page mais aussi toutes les ressources qu’il y a derrière, donc beaucoup de choses en général minimum 30 hits et jusqu’à 200 voire 300 si beaucoup de plugins

Tout ce processus là vous pouvez le voir dans l'onglet network dans vos navigateurs m'appuyant sur f12

Chaque hit est sauvegardée par le serveur dans un fichier de log ça c'est la configuration par défaut sur tous les serveurs web que ça soit du apache nginx insee etc tout est loggé par défaut donc ce qui veut dire pour une page vous allez avoir entre 30 et x lignes inscrites dans un fichier un fichier texte

### Quels sont les champs qui sont dans ce fichier de log ?

- le timestamp : l'heure et la date à laquelle le hit a été effectué
- l'adresse ip du client ou du robot
- l'url qui a été demandé
- le status code c'est le code de retour du serveur (200 tout s'est bien passé, 300 une redirection permanente, 302une réduction temporaire, 404 si la page introuvable, 500 pour une erreur serveurs etc etc)
- le useragent :l'identité déclarée du visiteur (c'est assez facilement modifiable)
