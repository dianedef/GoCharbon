# WeWeb

## NO-CODE : LE BUILDER FRANÇAIS DE WEB APPS FRONTEND CONNECTÉ À TON BACKEND

### tl;dr

WeWeb (weweb.io), c'est le **builder français de web apps no-code** — la plateforme qui te permet de créer des **applications web complexes** avec une interface visuelle drag & drop, sans écrire de code. La différence avec Bubble : WeWeb est un **frontend builder** qui se connecte à n'importe quel **backend** (Supabase, Xano, Airtable, API REST). Tu dessines ton interface visuellement, tu connectes tes données, tu ajoutes de la logique (workflows, conditions, boucles), et tu publies. C'est le Figma-to-app français. Fondé en **2021** à **Paris**. Cible : **startups, agences no-code, PME, équipes produit** qui veulent construire des apps web sur-mesure sans développeur.

### Alternative à

Bubble, Webflow (volet apps), Retool, FlutterFlow, Softr. WeWeb se distingue par son **approche frontend-agnostique** (tu choisis ton backend — pas de lock-in), sa **performance** (génère du code Vue.js natif, pas de runtime lourd comme Bubble), et sa **flexibilité** (CSS custom, composants personnalisés, code custom en JS).

## Bénéfices

**Frontend + backend découplé** — WeWeb gère le frontend (interface, design, interactions), tu connectes le backend de ton choix (Supabase, Xano, Firebase, REST API). Pas de vendor lock-in sur les données.

**Performance native** — WeWeb génère du code Vue.js. Les apps sont rapides et SEO-friendly, contrairement à Bubble qui est notoirement lent. Le code peut même être exporté.

**Design pixel-perfect** — Contrôle total sur le design : CSS natif, flexbox, grid, responsive. Tu peux reproduire n'importe quel design Figma, pas juste des templates prédéfinis.

**Logique avancée** — Workflows visuels avec conditions, boucles, variables, formules. Tu crées des apps complexes (CRUD, dashboards, portails) sans écrire de code.

**Extensible** — Tu peux ajouter du code custom (JS, CSS) quand le no-code ne suffit pas. Le meilleur des deux mondes.

## Comment Utiliser WeWeb

1. **Inscription** — Crée un compte gratuit sur [weweb.io](https://www.weweb.io/)
2. **Projet** — Crée un nouveau projet ou pars d'un template
3. **Design** — Construis ton interface en drag & drop (composants, layout, style)
4. **Données** — Connecte ton backend (Supabase, Xano, Airtable, API)
5. **Logique** — Ajoute les workflows (actions, conditions, navigation)
6. **Publish** — Publie ton app en un clic (hébergement inclus ou custom domain)

## Fonctionnalités

### Éditeur visuel

- **Drag & drop** — Composants glissés sur le canvas
- **Layout** — Flexbox et CSS Grid natifs
- **Responsive** — Breakpoints desktop, tablette, mobile
- **Components** — Bibliothèque de composants (formulaires, tableaux, modals, cartes, navbars)
- **Custom CSS** — Contrôle CSS complet pour chaque élément
- **Design system** — Variables de style (couleurs, typographies, espacements)

### Connecteurs de données

- **Supabase** — Connecteur natif (auth, database, storage)
- **Xano** — Connecteur natif (API backend no-code)
- **Airtable** — Connecteur natif
- **REST API** — N'importe quelle API REST/GraphQL
- **Firebase** — Auth et Firestore
- **Auth0** — Authentification
- **OpenAI** — Connecteur IA
- **Plugins** — Marketplace de plugins communautaires

### Logique et workflows

- **Workflows** — Séquences d'actions visuelles (on click → fetch → update → navigate)
- **Variables** — Variables locales, globales, de page
- **Conditions** — Affichage conditionnel, logique if/else
- **Boucles** — Itération sur les données (listes, tableaux)
- **Formules** — Formules et transformations de données
- **Événements** — On load, on click, on change, custom events

### Authentification

- **Auth native** — Login/signup avec email, OAuth (Google, GitHub)
- **Rôles** — Gestion des rôles et permissions
- **Pages privées** — Pages accessibles uniquement aux utilisateurs connectés
- **Middleware** — Redirections selon l'état de connexion

### Publication

- **Hébergement** — Hébergement inclus sur weweb.io
- **Custom domain** — Domaine personnalisé
- **SEO** — Métadonnées, sitemap, SSR pour le référencement
- **Export de code** — Possibilité d'exporter le code Vue.js généré
- **Staging** — Environnement de staging avant publication

## L'équipe et l'Histoire

- Création de **WeWeb** en **2021** à **Paris** par **Raphael Music** (CEO) et **Joyce Kettering** (CTO)
- L'idée : les outils no-code comme Bubble sont puissants mais créent un lock-in total (données, hébergement, performance). WeWeb propose l'inverse : un frontend builder performant qui se connecte à n'importe quel backend
- **Levée de fonds** : WeWeb a levé des fonds pour accélérer le développement de sa plateforme et son expansion internationale
- La communauté WeWeb grandit rapidement avec des agences no-code qui l'adoptent comme outil principal de delivery
- WeWeb est particulièrement populaire dans l'écosystème **Supabase + WeWeb** (le "stack no-code open-source")

## Cas d'Usage Concrets

**La startup (MVP en 2 semaines)** — Tu as une idée de marketplace. En 2 semaines, tu construis le frontend complet sur WeWeb (pages produits, recherche, panier, profils) connecté à Supabase (base de données, auth, storage). Tu lances ton MVP avec 0 ligne de code, et l'app est rapide parce que c'est du Vue.js natif.

**L'agence no-code (portail client)** — Un client veut un portail de suivi de commandes pour ses clients B2B. Tu crées l'interface sur WeWeb (dashboard, liste de commandes, détails, filtres), connectée à l'API du client (REST). En 3 semaines, le portail est en prod. Le client peut modifier le design lui-même.

**L'équipe produit (dashboard interne)** — Tu as besoin d'un dashboard interne pour visualiser les KPIs de ton SaaS. WeWeb connecté à Supabase : tableaux, graphiques, filtres, exports. En 1 semaine, ton dashboard est prêt — pas besoin d'attendre 3 sprints de développement.

## Points Forts

- Frontend builder no-code avec performance native (Vue.js)
- Pas de vendor lock-in (backend au choix)
- Design pixel-perfect (CSS natif, flexbox, grid)
- Connecteurs riches (Supabase, Xano, REST API, etc.)
- Code exportable
- Communauté active et marketplace de plugins
- Interface en français
- Développé en France

## Points Faibles

- **Courbe d'apprentissage** — WeWeb est plus puissant que Webflow ou Softr, mais aussi plus complexe. Comptez quelques jours de formation
- **Backend séparé** — Tu dois configurer ton backend toi-même (Supabase, Xano). Ce n'est pas un all-in-one comme Bubble
- **Mobile** — WeWeb fait des web apps responsive, pas des apps mobiles natives. Pour du natif, il faut FlutterFlow ou Adalo
- **Communauté** — Plus petite que Bubble (qui a 10 ans d'avance). Moins de templates et tutoriels disponibles
- **Prix** — Le plan gratuit est limité. Pour un usage professionnel, il faut un plan payant
