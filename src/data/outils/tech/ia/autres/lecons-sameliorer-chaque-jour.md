# Contenu

- **Description initiale :** La description initiale de l'application est cruciale, car elle guide l'IA dans la bonne direction.
- **Refactoring constant :** Le code doit souvent être révisé, nécessitant un mécanisme pour revenir et réévaluer les chemins pris pendant l'implémentation.
- **Examen par les agents :** Les agents peuvent se relire mutuellement pour améliorer la qualité du code.
- **Focus sur un problème :** L'IA fonctionne mieux lorsqu'elle se concentre sur un seul problème à la fois.
- **Journalisation et modularité :** Des loggs détaillés et une base de code modulaire facilitent le débogage et l'implémentation des fonctionnalités.
