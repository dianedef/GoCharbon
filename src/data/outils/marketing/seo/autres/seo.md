BERT, qui signifie "Bidirectional Encoder Representations from Transformers", est un modèle de traitement du langage naturel développé par Google. Il a été introduit en 2018 et a révolutionné la manière dont les machines comprennent le langage humain. Voici quelques caractéristiques clés de BERT :

1. **Bidirectionnalité** : Contrairement aux modèles précédents qui lisaient le texte de manière unidirectionnelle (de gauche à droite ou de droite à gauche), BERT lit le texte dans les deux directions simultanément. Cela lui permet de mieux comprendre le contexte des mots en tenant compte des mots qui les entourent.

2. **Transformers** : BERT utilise une architecture de type transformer, qui est particulièrement efficace pour traiter des séquences de mots. Cette architecture permet au modèle de se concentrer sur différentes parties du texte et de capturer des relations complexes entre les mots.

3. **Pré-entraînement et ajustement** : BERT est pré-entraîné sur une grande quantité de texte en utilisant des tâches telles que la prédiction de mots masqués et la classification de phrases. Après ce pré-entraînement, il peut être ajusté pour des tâches spécifiques, comme la classification de texte ou la réponse à des questions.

4. **Applications** : BERT est utilisé dans de nombreuses applications de traitement du langage naturel, y compris la recherche d'information, l'analyse des sentiments, et la réponse à des questions. Il a également amélioré la qualité des résultats de recherche sur Google.

En résumé, BERT est un modèle puissant qui a considérablement amélioré la compréhension du langage par les machines, en permettant une meilleure interprétation du contexte et des relations entre les mots.

    - Le SEO (search engine optimization) regroupe un ensemble d'actions que nous pouvons mettre en œuvre pour améliorer la visibilité d'un site sur un moteur de recherche. Parmi les leviers actionnables, les 3 principaux qui se devaient d'être réunis étaient :  
          
        \- Aucune erreur technique sur le site afin que les moteurs puissent y naviguer facilement ;  
        \- Un contenu minimal afin que le moteur puisse vaguement comprendre quelle était la thématique de votre site ;  
        \- Des liens afin de faire valoir aux moteurs que votre site était cité de partout, et donc, qu'il était populaire.
    - Les 2e et 3e leviers ont malheureusement fait l'objet d'un travail si intense de la part des référenceurs (netlinking notamment) qu'il a rapidement montré ses limites tant il était facile de le manipuler.         Si nous profitions de notre moteur pour étudier le comportement des internautes ? Nous avons là le plus énorme panel d'utilisateurs de la planète sur toutes les thématiques et nous n'avons même pas besoin de les rémunérer.         **Voyons comment cela pourrait se passer**  
        ------------------------------------------  
          
        Un internaute tape une requête. Mais comme il est fainéant, ce sera 2 ou 3 mots seulement. Par contre, il a un historique de recherche. Le moteur connaît aussi les sites qu'il a visités auparavant, ceux où il est resté, les pages qu'il est ensuite allé voir en cliquant sur les liens du site, mais aussi les sites dont il s'est enfui. Ceci particulièrement si le « si pratique » Chrome a été utilisé pour cette navigation… Derrière, un peu d'analyse du contexte des sites visités (textes, liens suivis ou pas, etc.) et Google peut alors modifier la requête de l'internaute, à son insu, pour lui fournir un résultat de recherche personnalisé. Cela revient à dire que la page de résultats ne correspond pas stricto sensu à ce que l'internaute a tapé, mais plutôt à ce qu'il cherche, selon Google. Et plus ça va, plus Google gagne en précision…  
          
        Du même coup, les pages doivent être optimisées sur ce que l'internaute cherche et pas sur ce qu'il tape. Cela change toute l'approche que nos devons avoir pour nos contenus. Nous nous sommes rendu compte de cela dès les premiers travaux de R&D sur les metamots il y a maintenant 5 ans. Cela n'a fait que s'étendre et se développer depuis lors.

Le web fonctionne sur le principe d’échanges décentralisés entre plusieurs parties.

On dit "décentralisé" car il n'y a pas une seule institution/entreprise/groupe qui régit les échanges, mais une infinité d'acteurs.

Les clients : Vous, en général, via votre navigateur web

Sauf qu'au lieu d'un plat de pâtes carbonara ils s'agit de fichiers qui corresponent à des pages web, mais le principe est le même. Le serveur va “servir” des pages web à votre navigateur, qui va vous les afficher.

En détails :

Pour accéder à un site web, vous rentrez l’adresse d’un site web (dont le nom savant est URL pour Uniform Resource Locator) dans votre navigateur. Par exemple, vous êtes actuellement sur l'URL : [https://openclassrooms.com/fr/courses/5489551-creez-un-site-moderne-et-professionnel-avec-wordpress/5804091-initiez-vous-au-fonctionnement-d-un-site-web](__MASK_15__)

On peut décomposer l’URL en plusieurs parties : “HTTP”, c’est le nom du protocole de communication entre le client et le serveur, ici : Hypertext Transfer Protocol, c’est le protocole le plus utilisé permettant d’afficher des pages web (c’est-à-dire de l’hyper text : du texte et des liens vers d’autres pages web). Il en existe d’autres pour d’autres usages, comme FTP (pour File Transfer Protocol) que vous connaissez peut-être et servant lui à transférer… des fichiers bien sûr ! 

techniquement ? Votre navigateur va demander sur le réseau la localisation du site [openclassrooms.com](http://openclassrooms.com). Le réseau va lui répondre sous la forme d’une IP, qui est en gros l’adresse du serveur sur le réseau, et va permettre à la requête d’être acheminée sur le serveur contenant le site web. Une fois la requête acheminée sur le serveur d’OpenClassrooms, le serveur va regarder le chemin d’accès de la page demandée et la fournir au format HTML. Le navigateur va enfin interpréter le HTML fourni pour créer la page web telle que vous la voyez sur votre écran ! Bref, pour disposer d’un site web en ligne, il vous faut donc un serveur et un nom de domaine ! Rentrons donc un peu plus dans le détail de ces deux concepts. Qu’est-ce qu’un hébergement web ?

Pour fonctionner, tout site internet s’appuie donc sur un serveur. En général on loue un serveur auprès d’un hébergeur, mais il est tout à fait possible de créer son propre serveur chez soi. Cela demande en revanche des compétences techniques qui vont largement au-delà de ce que l’on couvrira dans ce cours. Fonctionnement d'un site internet Le serveur stocke les fichiers qui composent le site web et exécute le code contenu dans ces fichiers. Dans le cas de WordPress, il s’agit d’un CMS reposant principalement sur le langage PHP et sur une base de données MySQL. Le serveur exécute donc le code PHP de WordPress pour récupérer les informations nécessaires en base de données et générer la page au format HTML, lisible par votre navigateur web (Google Chrome, Mozilla Firefox, etc.).
